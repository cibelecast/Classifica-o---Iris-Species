#Esta função train_test_split divide o conjunto de dados, em conjunto de dados de treino e conjunto de dados de teste.
#Divide-se o nosso dataset em conjunto de dados de treino para construir o modelo, e o outro conjunto de dados de teste
#é para avaliar se o modelo terá um bom desempenho com novos dados(*que não foram vistos antes).Desse modo, temos o  
#benefício de se ter dados de teste para avaliação do desempenho do modelo para evitar,por exemplo, o problema de 
#overfitting(erro de variância) dos dados. Se não houver esta divisão dos datasets, o seu modelo pode não ter precisão
#e passar a ser inadequado para a determinada solução, pois não seria capaz de prever novos dados, e por isso iria 
#falhar.
# O X deve ser entendido como os dados de entrada e Y deve ser entendido como os dados de saida. X_train y_train vão ser
#utilizados pelo classificador da função e X_teste e y_teste vão ser utilizados pelo parametro score para poder realizar
#a avaliação do modelo.

#uma classe de "iris" dentro desse modelo

#na metade, e ai isso não iria possibilitar do modelo realizar sua previsão, para mais detalhes leia o livro. Pois, não
#iria abranger todas as classes.
